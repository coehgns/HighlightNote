# HighlightNote 데모 스크립트

## 데모 목표

1. 사용자가 직접 표시한 하이라이트를 기준으로 노트가 생성된다는 점을 보여준다.
2. 실패 상태도 숨기지 않고 분리해서 처리한다는 점을 보여준다.

## 준비물

- 하이라이트가 포함된 PDF 1개
- 하이라이트가 없는 텍스트 PDF 1개
- 이미지 기반 PDF 1개
- 로컬 실행 환경 또는 EC2 배포 환경

## 데모 순서

### 1. 정상 플로우
1. 브라우저에서 HighlightNote 메인 화면을 연다.
2. 하이라이트가 포함된 PDF를 업로드한다.
3. 업로드가 완료되면 생성된 노트 초안과 source highlight를 보여준다.
4. `Download PDF` 버튼으로 노트를 저장한다.

### 2. 무하이라이트 플로우
1. 하이라이트가 없는 텍스트 PDF를 업로드한다.
2. 상태가 `NO_HIGHLIGHTS`로 노출되는지 확인한다.

### 3. 비지원 PDF 플로우
1. 이미지 기반 PDF를 업로드한다.
2. 상태가 `UNSUPPORTED_PDF`로 노출되는지 확인한다.

## 발표 포인트

- 이 서비스는 AI가 임의로 중요한 문장을 고르는 방식이 아니라, 사용자가 직접 표시한 하이라이트를 우선한다.
- 노트 섹션마다 원본 highlight 근거가 남아 있어 추적이 가능하다.
- 실패도 조용히 무시하지 않고 명확한 상태 코드와 메시지로 처리한다.
